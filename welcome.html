Programming Best Practices

How to learn Python and R the way professionals do. Welcome to the SI Analytics Bootcamp.

Why This Session Matters

Programming is not about memorizing syntax. It is about developing a way of thinking. The habits you form early in this bootcamp will determine whether you merely complete lessons or emerge as a confident practitioner.

You do not learn programming by watching code. You learn programming by writing, breaking, and fixing code.

The Illusion of Progress

Many learners consume programming content passively, watching tutorials, scrolling through notebooks, and running prepared scripts. It feels productive, but it is deceptive.

You are observing code, not internalizing it. Programming is a muscle skill, and it only grows through deliberate practice.

Best Practices That Work

  • Always code along when watching lessons.
  • Use two screens or split your screen whenever possible.
  • Type code yourself instead of copying and pasting.
  • Expect things to break. That is where learning happens.

The Learning Framework

Watch

Understand what the code is meant to do, not every detail.

Type

Manually type every line in your IDE. Avoid copy-paste.

Tweak

Change variables, functions, or datasets and observe what breaks.

Build

Create something small but meaningful using what you just learned.

Repeat

Consistency beats intensity. Two hours daily builds mastery.

From Learning to Thinking Like a Programmer

In data science and public health, code is a tool for turning messy data into insight. When you code actively, you train yourself to think logically, troubleshoot systematically, and communicate evidence effectively.

Start small. Code daily. Break things. Fix them. That is how confidence is built.
